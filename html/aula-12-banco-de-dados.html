🗄️ O Que é Banco de Dados?

Banco de dados é um jeito organizado de guardar informações no computador. Serve para armazenar, consultar e atualizar dados de forma rápida e segura. Está presente em quase tudo: sites, aplicativos, empresas, escolas e muito mais!

Características Principais:

Característica Descrição
Grande Volume Lida com grandes volumes de dados
Recursos Oferece recursos poderosos para gerenciar estrutura, integridade, segurança e acesso
Confiável Garante consistência e confiabilidade das informações

💡 Importante: Banco de dados é onde guardamos informações de forma organizada. Eles são usados em muitos lugares e ajudam a acessar, proteger e controlar dados com eficiência.

🔧 SGBD - Sistema de Gerenciamento de Banco de Dados

SGBD é um software que permite a criação, a definição, o armazenamento, a recuperação e a manipulação de dados em um banco de dados.

Principais SGBDs:

  • MySQL - Open source, muito popular
  • PostgreSQL - Open source, avançado
  • Oracle - Corporativo, robusto
  • SQL Server - Microsoft, enterprise
  • SQLite - Leve, embutido
  • MongoDB - NoSQL, documentos

Operações em um SGBD:

Operação Descrição
CREATE Acrescentar novos arquivos/tabelas
INSERT Inserir novos dados
SELECT Buscar dados existentes
UPDATE Alterar dados existentes
DELETE Eliminar dados ou arquivos

📊 Os Três Níveis do Banco de Dados

Os bancos de dados são projetados em três níveis de abstração:

Nível Nome Descrição
1 Conceitual Não contém detalhes sobre a representação física. Descreve O QUE guardar.
2 Lógico Descreve como as informações estão organizadas internamente. Visão do usuário.
3 Físico Descreve os dados no nível mais baixo. Aspectos de implementação do SGBD.

🎨 Modelo Conceitual

O Modelo Conceitual é a primeira etapa do projeto de banco de dados. É uma representação abstrata e de alto nível dos dados, sem se preocupar com detalhes técnicos.

Finalidade:

  • Descrever a estrutura de um banco de dados de forma independente de um SGBD
  • Facilitar a comunicação entre técnicos e não-técnicos
  • Documentar a lógica do negócio

Representação:

Diagrama Entidade-Relacionamento (DER)

Principais Conceitos do Modelo E-R:

Conceito Descrição Representação
Entidade Conjunto de objetos da realidade sobre os quais se deseja manter informações Retângulo com nome
Atributos Características das entidades Círculo ou elipse
Relacionamentos Associação entre entidades Losango ligado por linhas

Exemplos de Entidades:

Sistema Bancário:

  • Cliente
  • Conta Corrente
  • Conta Poupança
  • Agência

Sistema de Controle de Produção:

  • Produto
  • Empregado
  • Departamento
  • Estoque

Tipos de Atributos:

Tipo Descrição Exemplo
Compostos Podem ser divididos em subpartes Endereço → Rua, Cidade, Estado
Simples Não são divisíveis Idade, Nome
Monovalorados Valor único para uma entidade Idade, CPF
Multivalorados Conjunto de valores para a mesma entidade Titulação, Telefones

Chave (Identificador):

⚠️ Chave Primária: É um tipo de restrição de integridade que visa garantir que não possam existir duas ou mais entidades com os mesmos valores identificadores. Representada por um círculo preenchido.

Cardinalidade de Relacionamentos:

Cardinalidade Significado Exemplo
1:1 Um para um Pessoa ↔ CPF
1:N Um para muitos Departamento ↔ Funcionários
N:N Muitos para muitos Aluno ↔ Disciplina

📋 Modelo Lógico

O Modelo Lógico é a representação organizada das tabelas do banco de dados. Ele mostra nome das tabelas, campos, tipos de dados, chaves primárias e estrangeiras.

Características:

  • Dados organizados em linhas e colunas
  • Estrutura simples
  • Fácil consulta
  • Muito utilizado em bancos relacionais

Do Conceitual ao Lógico:

Modelo Conceitual Modelo Lógico
Entidade Tabela
Atributo Campo/Coluna
Relacionamento Chave Estrangeira
Identificador Chave Primária

Tipos de Dados Comuns:

Tipo Descrição Exemplo
INT Números inteiros idade, quantidade
VARCHAR Texto de tamanho variável nome, email
DECIMAL Números decimais exatos preço, salário
DATE Apenas data data_nascimento
DATETIME Data e hora data_cadastro
BOOLEAN Verdadeiro ou falso ativo, disponível

Chaves no Modelo Lógico:

Chave Descrição Características
Primary Key (PK) Identifica unicamente cada registro Não pode repetir, não pode ser nula, deve ser única
Foreign Key (FK) Cria relacionamento entre tabelas Referencia PK de outra tabela

Auto Incremento:

💡 AUTO_INCREMENT: O banco gera automaticamente o próximo número do campo id. Exemplo: Cliente Pedro → id 1, Cliente Caue → id 2, Cliente Ana → id 3

Relacionamento 1:N (Um para Muitos):

Exemplo: Um curso possui vários alunos.

Implementação: A FK fica no lado "muitos".

💾 Modelo Físico - MySQL

O Modelo Físico é a implementação concreta do banco de dados em um SGBD específico. Aqui transformamos o modelo lógico em código SQL real.

Acessando o MySQL pelo Terminal:

mysql -u root -p

Senha: senacrs

Comandos Básicos:

-- Ver bancos de dados existentes
SHOW DATABASES;

-- Criar um banco novo
CREATE DATABASE meu_banco;

-- Usar um banco específico
USE meu_banco;

-- Ver tabelas do banco atual
SHOW TABLES;

-- Ver estrutura de uma tabela
DESCRIBE nome_tabela;

-- Sair do MySQL
EXIT;

Exemplo Prático - Criando Tabelas:

CREATE DATABASE loja_online;
USE loja_online;

CREATE TABLE cliente (
  id_cliente INT AUTO_INCREMENT PRIMARY KEY,
  nome VARCHAR(100) NOT NULL,
  email VARCHAR(150) UNIQUE NOT NULL,
  telefone VARCHAR(15)
);

CREATE TABLE produto (
  id_produto INT AUTO_INCREMENT PRIMARY KEY,
  nome VARCHAR(100) NOT NULL,
  preco DECIMAL(10, 2) NOT NULL,
  estoque INT DEFAULT 0
);

CREATE TABLE pedido (
  id_pedido INT AUTO_INCREMENT PRIMARY KEY,
  data_pedido DATETIME DEFAULT CURRENT_TIMESTAMP,
  valor_total DECIMAL(10, 2),
  id_cliente INT NOT NULL,
  FOREIGN KEY (id_cliente) REFERENCES cliente(id_cliente)
);

Inserindo Dados:

INSERT INTO cliente (nome, email, telefone)
VALUES ('João Silva', 'joao@email.com', '51999887766');

INSERT INTO produto (nome, preco, estoque)
VALUES ('Notebook Dell', 2999.90, 10);

INSERT INTO pedido (valor_total, id_cliente)
VALUES (2999.90, 1);

Consultando Dados:

-- Ver todos os clientes
SELECT * FROM cliente;

-- Ver apenas nome e email
SELECT nome, email FROM cliente;

-- Ver produtos com preço acima de 100
SELECT nome, preco FROM produto WHERE preco > 100;

-- Ver pedidos com nome do cliente (JOIN)
SELECT p.id_pedido, c.nome AS cliente, p.valor_total
FROM pedido p
INNER JOIN cliente c ON p.id_cliente = c.id_cliente;

📝 Exercícios Práticos

Questão 1 - Academia

Desafio: Crie um banco de dados chamado academia com:

  1. Insira ao menos 4 alunos
  2. Insira ao menos 3 planos diferentes
  3. Insira ao menos 5 matrículas, com ao menos 2 já vencidas (data_fim no passado)
  4. Faça uma consulta que mostre todos os alunos e todos os campos
  5. Faça uma consulta nas matrículas mostrando apenas data_inicio e data_fim
  6. Delete as matrículas com data_fim anterior a 2024
  7. Altere o e-mail de 1 aluno

Questão 2 - Música

Desafio: Crie um banco de dados chamado musica com:

  1. Insira ao menos 3 compositores e 3 estilos
  2. Insira ao menos 5 músicas, cada uma vinculada a um estilo
  3. Faça uma consulta mostrando todos os campos da tabela de músicas
  4. Faça uma consulta mostrando apenas o título e a duração das músicas
  5. Altere o estilo de 1 música
  6. Delete 1 compositor e seus vínculos com as músicas

Questão 3 - Projeto Livre

Desafio: Crie um banco de dados com nome à sua escolha, com pelo menos 3 tabelas relacionadas:

  1. Insira ao menos 3 registros em cada tabela
  2. Faça uma consulta em cada tabela
  3. Altere 2 registros em tabelas diferentes
  4. Delete 1 registro de 2 tabelas diferentes
