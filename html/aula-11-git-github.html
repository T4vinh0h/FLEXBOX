📖 O que são Git e GitHub?

Nome Definição Função
Git Sistema de controle de versão Salva histórico de alterações
GitHub Plataforma online Armazena e compartilha repositórios

Por que usar?

  • ✅ Salva histórico completo do projeto
  • ✅ Volta para versões antigas
  • ✅ Trabalha em equipe sem conflitos
  • ✅ Portfólio de projetos públicos
  • ✅ Backup na nuvem

⚙️ Configuração Inicial

1. Criar conta no GitHub:

Acesse: https://github.com

2. Instalar Git:

# Windows
https://git-scm.com/download/win

# Linux (Ubuntu/Debian)
sudo apt install git

# Verificar instalação
git --version

3. Configurar Git (OBRIGATÓRIO):

⚠️ Importante: Use o MESMO email da sua conta GitHub!
git config --global user.name "Seu Nome"
git config --global user.email "seu@email.com"
git config --list /* Ver configuração */

📚 Comandos Essenciais

Comando Descrição Quando usar
git init Inicia repositório Ao começar projeto novo
git status Ver estado Para ver mudanças
git add . Adiciona tudo Antes de commit
git commit -m "msg" Salva versão Para salvar mudanças
git push Envia para GitHub Para salvar na nuvem
git pull Baixa atualizações Antes de trabalhar
git clone URL Copia repositório Baixar projeto

🔄 Fluxo de Trabalho Diário

Passo Comando Descrição
1 git pull Baixa atualizações
2 Trabalha Faz alterações
3 git status Ver o que mudou
4 git add . Prepara tudo
5 git commit -m Salva localmente
6 git push Envia para GitHub

🌳 Branches (Ramificações)

Comando Ação
git branch Lista branches
git branch nome Cria branch
git checkout nome Troca de branch
git merge nome Junta branches

🎨 Ferramentas Visuais

GitHub Desktop:

  • Interface gráfica (sem comandos)
  • Download: https://desktop.github.com/
  • Gerencia branches visualmente

Extensão Git do VSCode:

  • Ícone "Source Control" na barra lateral
  • Veja alterações em tempo real
  • Clique "+" para git add
  • Escreva mensagem e clique em "✓" para commit

✅ Exercício Prático

🎯 Desafio:

  1. Crie conta no GitHub
  2. Instale Git e configure nome/email
  3. Crie repositório no GitHub
  4. Clone para computador
  5. Crie arquivo index.html
  6. git add, commit, push
  7. Veja no GitHub!
