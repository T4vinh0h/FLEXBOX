📖 Tags Semânticas HTML5

HTML5 introduziu tags semânticas que dão significado ao conteúdo, tornando o código mais organizado e compreensível. Em vez de usar <div> para tudo, podemos usar tags que explicam o propósito.

Por que usar?

Vantagem Explicação
Código organizado Tags explicam o propósito do conteúdo
Melhor SEO Buscadores entendem a estrutura
Acessibilidade Leitores de tela navegam melhor
Manutenção Código mais fácil de manter

🔝 Tags Semânticas Principais

header, main, footer:

Tag Função O que vai dentro
<header> Cabeçalho da página ou seção Logo, menu, título
<main> Conteúdo principal (único na página) Artigos, produtos
<footer> Rodapé da página Contato, copyright, links

Exemplo de Estrutura:

<body>
  <header>Cabeçalho</header>
  <main>Conteúdo principal</main>
  <footer>Rodapé</footer>
</body>

🔄 CSS Reset

Os navegadores aplicam estilos padrão. O CSS Reset remove esses estilos para começar do zero.

Reset Básico:

* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}
💡 Por que box-sizing: border-box? Com este valor, o tamanho que você define é o tamanho TOTAL (incluindo padding e border).

📏 Unidades de Medida CSS

CSS oferece várias unidades para definir tamanhos. Cada uma tem seu propósito e comportamento específico.

Unidade Tipo Baseado em Quando usar
px Fixo Pixel da tela Bordas, sombras, precisão
% Relativo Elemento pai Larguras responsivas
vh Relativo Altura da tela Seções fullscreen
vw Relativo Largura da tela Elementos fullwidth
rem Relativo Font-size do html Fontes globais
em Relativo Font-size do pai Componentes

✅ Exercício Prático

🎯 Desafio:

  1. Crie página com header, main, footer semânticos
  2. Adicione CSS Reset
  3. Use rem para fontes
  4. Use % para larguras responsivas
  5. Experimente vh para seção fullscreen
